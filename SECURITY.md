### Data Protection
- **End-to-End Encryption**: All user data is encrypted before storage
- **Local-First Storage**: Data remains on the user's device by default
- **Secure Authentication**: Biometric and PIN-based app locking
- **HTTPS Only**: All communications use secure protocols

### Application Security
- **Content Security Policy**: Strict CSP headers implemented
- **Input Validation**: All user inputs are validated and sanitized
- **XSS Protection**: Cross-site scripting prevention measures
- **CSRF Protection**: Cross-site request forgery protection

### Privacy by Design
- **Zero Data Collection**: No unnecessary data collection
- **User Control**: Users have full control over their data
- **Transparent Processing**: Clear privacy policies and data handling
- **Right to Deletion**: Complete data deletion on user request

3. Send details to our security team via email or WhatsApp

When reporting a vulnerability, please include:

- Steps to reproduce
- Potential impact
- Your contact information for follow-up
- Any suggested fixes (optional)

- Review app permissions periodically
